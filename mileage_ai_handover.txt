2023~2025년 마일리지 계산 SHP 방문 결과입력.xlsx 인수인계 문서

목적
- 이 문서는 다른 AI가 "2023~2025년 마일리지 계산 SHP 방문 결과입력.xlsx" 파일을 이어서 다룰 때 반드시 이해해야 하는 입력 규칙, 거리(km) 해석 규칙, 예외 처리 규칙을 정리한 것이다.
- 이 파일의 핵심 목적은 사용자가 업무상 방문한 이동 내역을 RAW 시트에 기록하고, 월별 총 이동거리 합계를 파악하는 것이다.

파일의 기본 구조
- 주요 시트는 다음 2개다.
- RAW: 실제 원장 입력 시트
- 매월 합계: 월별 이동 내역/합계를 따로 정리한 시트

중요 전제
- 이 파일의 가장 중요한 목적은 "하루 단위 방문 내역 기록"과 "월별 총 이동거리 합산"이다.
- 사용자가 새로운 방문 일정을 알려주면 RAW 시트 형식으로 입력하면 된다.
- 거리(km)는 임의 계산하면 안 된다.
- 거리(km)는 반드시 기존 파일 안에 이미 존재하는 데이터를 우선적으로 참조해야 한다.
- 기존 파일 안에서 기준이 되는 거리값을 찾을 수 없으면 km 셀은 빈칸으로 둔다.

RAW 입력 기본 형식
- 보통 하루 방문은 왕복 2행으로 입력한다.
- 예:
  2026년 | 4월 | 27일 | Home | 포항시청,보건소 | 106
  2026년 | 4월 | 27일 | 포항시청,보건소 | Home | 106

기본 입력 원칙
1. 사용자가 하루에 Home에서 출발해 방문 후 Home으로 복귀한 경우:
- 1행은 "Home -> 방문지목록"
- 2행은 "방문지목록 -> Home"

2. 사용자가 당일 귀가하지 않고 숙박 이동이 있는 경우:
- 이동 구간을 끊어서 별도 행으로 입력한다.
- 예:
  Home -> 첫 방문지
  첫 방문지 -> 숙소이동
  숙소 -> 다음날 방문지
  마지막 방문지 -> Home

3. 거리(km)는 각 구간별로 개별 판단한다.
- Home 출발 구간은 Home -> 해당 지역 기준값 사용
- 귀가 구간은 해당 지역 -> Home 기준값 사용
- 숙소이동, 숙소 출발 구간은 파일 안에 기존 대응값이 있을 때만 사용
- 없으면 공란

절대 하면 안 되는 것
- AI가 지도 기준이나 추정치로 새 km를 계산해서 넣으면 안 된다.
- 인터넷 검색 등 외부 기준으로 거리값을 채우면 안 된다.
- 파일 안의 기존 패턴이 없는데 임의로 숫자를 넣으면 안 된다.

거리(km) 참조의 핵심 규칙
- 거리 참조는 "문자열 전체 완전일치" 기준이 아니다.
- 방문지 목록의 "가장 앞 장소"를 기준점으로 삼는다.
- 가장 앞 장소가 같으면 뒤에 방문지가 더 붙어도 그 지역의 기존 거리값을 가져올 수 있다.

예시 1: 포항시청 기준
- 사용자가 "포항시청,보건소,의회"라고 말해도
- 가장 앞 장소가 포항시청이면
- 기존 "포항시청" 계열 거리값을 우선 적용한다.
- 따라서 다음처럼 쓸 수 있다.
  Home -> 포항시청,보건소,의회 = 106
  포항시청,보건소,의회 -> Home = 106

같은 곳으로 간주하는 규칙
- 첫 장소 이름이 완전히 동일하지 않아도, 사용자가 같은 곳으로 간주하라고 정한 경우는 같은 지역 기준값을 쓴다.
- 즉 단순 텍스트 일치보다 사용자의 실무 규칙이 우선한다.

대표적인 동일 지역 간주 예시
- 경주시청 = 경주시의회
- 진천군청 = 진천군보건소 = 진천군의회 계열
- 포항시청과 포항 관련 보건소/의회 계열은 첫 기준점이 포항시청이면 같은 기준으로 처리 가능

예시 2: 경주
- 사용자가 "경주시청,보건소,의회,농협"이라고 말한 경우
- 파일 내 기존값이 "경주시의회,보건소,농협"으로 존재하면
- 경주시청과 경주시의회를 같은 곳으로 간주해서 같은 km를 사용한다.
- 예:
  Home -> 경주시청,보건소,의회,농협 = 81
  경주시청,보건소,의회,농협 -> Home = 81

예시 3: 진천
- 파일 안에 "진천군청,보건소,의회" 기록이 있으면
- 사용자가 "진천군보건소"라고만 말해도
- 진천 계열 같은 지역으로 보고 기존값을 쓴다.
- 예:
  Home -> 진천군보건소 = 210
  진천군보건소 -> Home = 210

왕복 거리가 항상 같은 것은 아님
- 어떤 지역은 갈 때/올 때 km가 다를 수 있다.
- 따라서 Home -> 지역, 지역 -> Home 값을 각각 따로 봐야 한다.

예시 4: 부산광역시청, 해운대구보건소
- 기존 데이터에
  Home -> 부산광역시청,소방본부,해운대보건소 = 135
  부산광역시청,소방본부,해운대보건소 -> Home = 128
- 이런 기록이 있다면
- 사용자가 "부산광역시청, 해운대구보건소"라고 말했을 때
  Home -> 부산광역시청,해운대구보건소 = 135
  부산광역시청,해운대구보건소 -> Home = 128
- 로 처리할 수 있다.

숙박 이동이 있는 일정 처리 규칙
- 숙박 일정은 하루 왕복으로 합치면 안 된다.
- 이동 단위를 끊어서 입력해야 한다.

예시 5: 1박 일정
- 사용자가
  3월 2일 Home에서 부산해운대구 방문
  3월 2일 당일 Home 복귀 안 함, 숙소로 이동
  3월 3일 숙소에서 부산시청 방문
  마지막에 Home 복귀
- 라고 말하면 다음처럼 구간을 나눈다.

  2026년 | 3월 | 2일 | Home | 부산해운대구 | 135
  2026년 | 3월 | 2일 | 부산해운대구 | 숙소이동 |
  2026년 | 3월 | 3일 | 숙소 | 부산시청 |
  2026년 | 3월 | 3일 | 부산시청 | Home | 128

- 여기서 빈칸인 이유는 파일 안에 해당 숙소 이동 기준값이 없기 때문이다.

매우 중요한 실무 규칙
- "가장 비슷한 기존 행 하나를 찾아 그대로 쓰는 방식"만으로는 실제 사용자의 규칙을 완전히 재현할 수 없다.
- 같은 지역이라도 파일 안에 후보 km가 여러 개 존재할 수 있다.
- 이때 사용자는 단순 문자열 매칭이 아니라 "실무상 대표값" 또는 "같은 곳으로 간주하는 운영 기준"으로 km를 넣고 있다.

즉, 다른 AI가 반드시 이해해야 할 점
- 파일 안에 같은 지역에 대해 여러 km 후보가 있을 수 있다.
- 이 경우 무조건 첫 검색값을 쓰면 안 된다.
- 사용자가 실제로 어떤 값을 표준처럼 쓰는지 확인해야 한다.
- 사용자의 판단이 파일의 단순 텍스트 일치보다 우선한다.

실제 대화에서 확인된 중요한 사례

1. 포항 계열
- "포항시청,보건소"는 106 사용
- "포항시청,보건소,의회"도 첫 기준점이 포항시청이면 106 사용 가능

2. 경주 계열
- "경주시청,보건소,의회,농협"은
- 기존 "경주시의회,보건소,농협" 기록을 같은 곳으로 간주해 81 사용

3. 부산광역시청/해운대 계열
- 갈 때 135, 올 때 128
- 왕복 동일값으로 처리하면 안 된다

4. 진천 계열
- "진천군청,보건소,의회" 기존 기록 존재
- "진천군보건소" 단독 표현이어도 같은 지역 기준으로 210 사용 가능

실제 검토 과정에서 드러난 주의점
- 이 엑셀은 한글 문자열이 도구에 따라 깨져 보일 수 있다.
- 따라서 단순 문자열 검색 결과가 비어 있다고 해서 실제 데이터가 없는 것으로 단정하면 안 된다.
- 특히 "진천" 사례에서 실제 데이터가 있었는데 초기 검색이 실패한 적이 있다.
- 그러므로 다른 AI는 검색 실패 시 반드시 다음을 추가 확인해야 한다.
  1. 주변 행 수동 검토
  2. 같은 지역의 군청/시청/의회/보건소 변형 확인
  3. 한글 인코딩 문제 가능성 의심

사용자의 실제 운영 방식에 대한 매우 중요한 이해
- 사용자는 단순히 "문자열이 같은가"만 보지 않는다.
- 사용자는 지역 단위로 같은 곳인지 판단한다.
- 사용자는 군청/시청/의회/보건소가 섞여 있어도 동일 생활권/방문권역으로 간주할 수 있다.
- 사용자는 어떤 경우 대표값처럼 특정 km를 선택한다.
- 따라서 다른 AI는 거리값을 기계적으로 채우지 말고, 항상 "이 지역을 사용자는 같은 곳으로 보고 있는가?"를 먼저 생각해야 한다.

다른 AI가 반드시 따라야 하는 의사결정 순서
1. 사용자가 알려준 일정의 각 행을 구간 단위로 분해한다.
2. 각 구간의 첫 장소를 확인한다.
3. 파일 안에서 해당 첫 장소 또는 같은 지역으로 간주되는 기존 기록을 찾는다.
4. 같은 지역 후보가 여러 개면, 사용자가 기존에 실제로 채택한 대표값 패턴이 있는지 본다.
5. 갈 때와 올 때는 별도 값으로 판단한다.
6. 숙소/숙소이동 구간은 별도 값이 있는지 확인한다.
7. 기준값을 찾지 못하면 공란으로 둔다.
8. 절대 임의 계산하지 않는다.

다른 AI가 사용자에게 확인해야 하는 상황
- 같은 지역 후보 km가 여러 개 있는데 어느 값을 대표값으로 써야 할지 불명확할 때
- 같은 곳으로 간주 가능한지 애매할 때
- 숙소이동 기준이 파일에 없고, 사용자가 별도 운영 기준을 갖고 있을 가능성이 있을 때

이미 대화에서 합의된 핵심 규칙 요약
- 새로운 일정은 RAW 시트 형식으로 입력한다.
- 거리(km)는 파일 내부 기존 데이터를 우선 사용한다.
- 기존 데이터가 없으면 공란으로 둔다.
- 가장 앞 장소를 기준으로 거리 참조한다.
- 뒤에 방문지가 더 붙어도 첫 장소가 같으면 같은 기준을 쓸 수 있다.
- 시청/군청/의회/보건소가 달라도 사용자가 같은 곳으로 간주하라고 하면 같은 곳으로 본다.
- 왕복은 같은 값일 수도 있고 다를 수도 있다.
- 숙박 일정은 구간을 쪼개서 입력한다.
- AI는 절대 새 거리값을 추정하지 않는다.

마지막으로
- 이 파일은 단순 데이터 입력 파일이 아니라 사용자의 실제 이동 패턴과 실무 판단이 누적된 파일이다.
- 따라서 다른 AI는 "정확한 문자열 일치"보다 "사용자의 지역 해석 방식"을 더 중요하게 봐야 한다.
- 이 문서에 적힌 규칙을 우선 적용하고, 애매하면 사용자의 기존 입력 결과를 다시 확인하는 방식으로 접근해야 한다.
